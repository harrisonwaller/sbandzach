# Privacy Notes — what must never go live

This site is **public**. No password, no login. Anyone with the link (and search
engines) can see everything that ships. Treat it accordingly.

## The rule

If a detail would help a stranger show up uninvited, find someone's home, or
reach a vendor directly — it does not belong here. Those logistics live on Zola
(behind their RSVP) and in group texts, not on this keepsake.

## Never put on this site

- **Planner / vendor phone numbers or emails** (coordinator, photographer, DJ…)
- **Private residences or rental addresses** — Airbnb / VRBO / family homes
- **Shuttle and transport rosters**, pickup points, or schedules
- **Flight details / arrival times** for the couple or guests
- **Room block details, confirmation numbers, door codes, gate codes**
- **Guest contact information** of any kind (phone, email, address)
- **Anything financial** — registry card numbers, Venmo/Zelle handles
- **Children's full names / schools**, or anyone's home location

## What is fine (and already here)

- The couple's first names and the wedding date.
- **Public venue names and their public addresses** — The Ritz-Carlton at Lake
  Oconee and the Great Waters Course at Reynolds Lake Oconee. These are
  businesses whose addresses are already listed publicly (including on Zola).
- General attire notes (e.g. "Black-Tie Optional").
- Photographs the couple have approved.

## The Vault is not private storage

Typing `060626` reveals the Vault. This is an *Easter egg*, **not** security —
its contents are in the public bundle and discoverable. Only put things there
that you'd be comfortable being seen if someone guessed the date. Never put true
secrets, anyone's private information, or sensitive media in the Vault.

## Before each deploy — 30-second check

1. Skim any new `meta.json` and `content/letters.ts` for names, phone numbers,
   or addresses that shouldn't be public.
2. Make sure nothing from a "logistics" document got pasted into a caption.
3. Confirm new photos are ones the couple approved for a public page.

## Current status

As of the last review, the site contains **no private logistical content** —
only the couple's names/date, public venue addresses, approved engagement
photographs, and clearly-marked forthcoming placeholders.
