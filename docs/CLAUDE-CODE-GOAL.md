# /goal — SBANDZACH.COM TIME CAPSULE

## CONTEXT

Build a public, luxury editorial wedding time-capsule website for Sara Beth Rountree & Zachary Copenhaver. Wedding date: June 6, 2026, Lake Oconee, Georgia. Their existing Zola site at `zola.com/wedding/sarabethandzachary` handles RSVP / registry / travel / FAQ. This site does NOT replace Zola — this is a *gift* and a *keepsake*, not a utility site.

The audience is Sara Beth and Zachary themselves. Their family and close friends are secondary visitors. The tone should feel like Vogue feature + private family archive + cinematic memory book. Quiet luxury. Editorial. Southern lake wedding.

Domain: `sbandzach.com` (will be attached via Vercel Domains after first deploy).

A working static HTML preview already exists in this repo at `/reference/sbandzach-preview.html`. Use it as the **visual starting point and aesthetic reference**, but do not blindly port it — turn it into a production app with proper architecture.

---

## ABSOLUTE NON-NEGOTIABLES

- Public site. No password, no login, no auth.
- It must look like they paid a million dollars for it. Anything less is failure.
- Mobile must be as beautiful as desktop. Not "responsive" — *custom*.
- No Squarespace energy. No generic Tailwind blocks. No cliché wedding tropes.
- No countdown timers. No confetti. No cartoon icons. No cheesy copy.
- No private logistical details (planner phone, Airbnb address, shuttle rosters, etc.) — even if they exist in repo files.
- Run the iteration loop in Section 6 until stopping criteria are met. Minimum 5 iterations. Do not exit early.

### Build script: `npm run media:build`
- Scans all folders under `/public/media/`
- For each file: detect type, extract width/height, generate blur placeholder via `plaiceholder`, generate thumbnail for videos
- Reads optional `meta.json` next to each file for caption/speaker/pullQuote
- Writes `/content/generated-media.json`
- Logs clear console summary: `Found 47 photos, 12 videos, 8 audio, 3 letters`
- Idempotent — safe to run repeatedly
- Does not break on empty folders

## 4. CHAPTERS / PAGES

Single-page scroll architecture. Each chapter is a section with its own anchor.

### Hero
- Full viewport, cream background
- Massive `Italiana` typography for names
- Date in roman numerals, location below
- Loading screen ("S · Z" mark) fades to hero on first visit
- Refined entrance motion (staggered fade-up, ~3 seconds total)
- Top bar slides away on scroll down, reappears on scroll up

### Featured Image
- Full-bleed engagement photo (use the strongest one — likely the columned monument shot)
- Subtle caption bottom-left in italic small caps

### Chapter I — In Frames (gallery)
- Engagement photos in editorial asymmetric grid
- Mix of aspect ratios and sizes — no uniform grid
- Click any photo → elegant lightbox (dark backdrop, navigation arrows, ESC to close)
- All photos served as Next.js `<Image>` with blur placeholders
- Mobile: 2-column grid with one full-width feature row per cluster

### Chapter II — The Weekend (schedule)
- Dark ink background (#1a1612), cream text
- Editorial timeline, NOT a card grid
- Treat like a printed wedding program
- Roman numerals for days

### Chapter III — The Place (venue)
- Two-column asymmetric layout: image left, copy right (alternating on second venue)
- Short, restrained venue copy about Reynolds Lake Oconee and the Ritz-Carlton
- One pull-quote treatment per venue

### Chapter IV — The Voices (THE EMOTIONAL CENTERPIECE)
**This is the most important chapter. Build it as future-ready scaffolding now, since real content arrives after the wedding.**

Layout pattern: each speech/toast/dance is its own viewport-sized "track" with this exact structure:

1. **Pull-quote first** — a single line from the speech, 5xl italic serif, white on near-black background, centered, generous margins
2. **Speaker attribution** below quote — small caps, gold accent: `BEST MAN — CADE`
3. **Play button** — minimal thin circle, no labels
4. **On play**:
   - Audio starts via wavesurfer.js (waveform in gold/cream)
   - Speaker's photo emerges as a 40% edge-bleed on one side (alternating left/right per track)
   - Full transcript scrolls below, with current line highlighted in gold
5. **Navigation**: thin progress markers at top showing "Track 1 of 7" + arrow keys to move between tracks
6. **Dances** use same template but with video playing as ambient texture instead of photo

Build the component with full functionality now. Stub it with placeholder content (one example track with placeholder text + audio) so it's testable. Real content (best man, father of the bride, mother-son dance, etc.) added later via `media.json`.

### Chapter V — Letters
- Wall of shorter quotes from people who didn't give a speech
- Asymmetric editorial layout, mixed sizes — looks like a Vanity Fair pull-quote spread
- Each card: italic quote + speaker attribution in small caps
- Mobile: single column, varies in size

### Chapter VI — The Archive
- A second-tier gallery of *every* approved photo
- Filter chips: Friday | Ceremony | Cocktail | Reception | After Party
- Click any → lightbox with adjacent navigation
- This is the "go deep" section — secondary navigation

### Chapter VII — Forthcoming (before-wedding placeholder)
- Elegant "coming soon" treatment for sections that fill in post-wedding
- Cards: Photographs, Voices, Letters, The Dance Floor, The Lake, The Toasts, The After Party
- After wedding, this section is removed/replaced

### Hidden — The Vault
- Easter egg: typing `060626` anywhere on the site reveals an additional chapter
- Contains the rawer/private content (family-only stuff later, candid moments)
- Visual treatment: handwritten-style intro line "Some things stayed quiet."
- Not linked anywhere — pure discovery moment

### Footer
- Dark ink. The "S B · Z" mark. "For all the days along the way." Roman numeral date. That's it.

## 5. AESTHETIC RUBRIC (the bar you're iterating toward)

Score every iteration against these 8 dimensions, 0-10 each. The site is not done until all are ≥9.

| Dimension | What 10/10 looks like |
|---|---|
| **Typography** | Hierarchy is unmistakable. Letter-spacing on caps is dialed in (≥0.3em). Italic usage is intentional, not decorative. Display/body pairing feels editorial. Line-height is generous on body copy (1.6–1.8). |
| **Spatial composition** | Asymmetric where intentional, symmetric where called for. Whitespace is generous and structural. No "Tailwind grid of three" energy. Vertical rhythm between sections is consistent. |
| **Color** | Cream / ink / honey palette with restraint. Gold is rare and intentional — never decorative. Photos do the color work. No purple gradients, no random color pops. |
| **Motion** | Used at high-impact moments only: loader, hero entrance, image hover, audio play. No micro-animations on every element. Easing curves are slow and refined (700ms+, custom cubic-bezier). |
| **Mobile** | Hero typography sized appropriately for narrow viewport. Gallery aspect ratios re-thought for mobile, not just shrunk. Type sizes scale via `clamp()`. Tap targets ≥44px. No horizontal scroll. |
| **Emotional resonance** | Their names hit hard on first load. The Voices chapter pull-quotes land. The schedule feels like a program card. The Vault feels like a secret. |
| **Performance** | Lighthouse mobile ≥90 (perf, accessibility, best practices). Images lazy-loaded below fold. Fonts preloaded. No layout shift. First image < 200KB. |
| **Copy** | No corporate language. No "celebrate their journey" clichés. Short, elegant, human. Roman numerals on dates. Lowercase italic accents. |

## 6. THE ITERATION LOOP (run this autonomously, do not exit early)

### Setup (one-time, before loop starts)
1. Install Playwright: `npm i -D playwright` then `npx playwright install chromium`
2. Create `/scripts/audit.ts` that screenshots key views at 1440x900 and 390x844 (iPhone 14):
   - Hero (top of page)
   - Featured image
   - Gallery mid-scroll
   - Schedule
   - The Voices (one track expanded)
   - Letters
   - Forthcoming
   - Footer
3. Save screenshots to `/audit/iter-{N}/{view}-{viewport}.png`
4. Create `/audit/SCORECARD.md` to track scores across iterations

### The loop (minimum 5 iterations, maximum 12)

For each iteration `N`:

**Step A — Snapshot.** Run `npm run audit:screenshot` to capture all views.

**Step B — Score.** For each of the 8 rubric dimensions, score 0-10. Write to scorecard with one-sentence justification per dimension.

**Step C — Identify lowest-scoring dimensions.** Pick the 2-3 lowest.

**Step D — Adopt a persona for the fix pass.** Step into one of these roles and audit through that lens:
- **Art Director** — typography, spatial composition, color, hierarchy
- **Mobile Designer** — narrow viewport quality, type scale, image crops
- **Motion Designer** — easing, timing, sequencing, restraint
- **Copy Editor** — every word, every label, every roman numeral
- **Performance Reviewer** — image sizes, font loading, layout shift
- **Privacy Reviewer** — confirm zero private logistical content
- **Mom Preview Reviewer** — would Beth (mother of bride) be moved by this? Is anything cringe?

**Step E — Fix.** Make focused changes to address the lowest scores. Do not introduce regressions — keep a `DO NOT REGRESS` list at top of scorecard and check against it.

**Step F — Commit.** `git commit` with message `iter-{N}: {persona} — {summary of changes}`.

**Step G — Repeat.** Go to Step A.

### Micro-detail checklist (run every iteration after iteration 3)

These specific items must all be addressed before declaring done:

- [ ] Letter-spacing on all uppercase headers ≥0.3em
- [ ] Italic accents are `Cormorant Garamond` italic, not faux-italic on display font
- [ ] All photos have blur placeholders, no layout shift on load
- [ ] Hero name typography breathes — line-height 0.95 or less
- [ ] Roman numerals used everywhere date appears
- [ ] Top bar transition is smooth, no jump
- [ ] Loading screen "S · Z" mark is centered and properly tracked
- [ ] Gallery has at least 3 different aspect ratios in play, not a uniform grid
- [ ] Schedule uses ink background, not cream
- [ ] Voices play button is a thin circle, not a filled chunky button
- [ ] Audio waveform color is gold/honey, not blue/default
- [ ] Transcript highlight follows audio playback
- [ ] Pull quotes are 5xl+ on desktop, scale with `clamp()`
- [ ] Mobile hero names fit comfortably without overflow
- [ ] Mobile gallery is not just desktop crammed down — re-laid-out
- [ ] Vault easter egg works — typing `060626` reveals it
- [ ] Footer is dark ink with cream text
- [ ] No focus rings are default browser blue — custom subtle outline
- [ ] No element uses default Tailwind colors (gray-500, etc.) — only from the project palette
- [ ] Smooth scrolling enabled but reasonable (no overshoot)
- [ ] Page passes Lighthouse mobile ≥90 on perf and accessibility

### Stopping criteria (ALL must be true)

1. All 8 rubric dimensions scored ≥9/10
2. Micro-detail checklist 100% complete
3. Minimum 5 iterations completed
4. Last iteration produced no score improvement (sign of convergence)
5. Privacy review passed — zero private content leaked
6. Mobile screenshots reviewed side-by-side with desktop and feel intentional
7. Mom Preview persona pass concluded "this would move Beth"

If at iteration 12 the criteria are still not met, STOP, write `/audit/UNRESOLVED.md` listing remaining issues, and surface to Harrison.

## 8. WHAT NOT TO BUILD

- Anything from `/uploaded_files` that contains private logistics (planner phone numbers, Airbnb addresses, shuttle rosters, flight arrival times)
- Password protection of any kind
- RSVP form, registry, FAQ, travel info — Zola owns these
- A countdown timer
- Stock photography of any kind
- Generic wedding tropes (rings, bells, hearts, cake icons)
- Squarespace-style parallax
- Hover effects that scale by more than 1.03
- More than two display fonts
- Any text in Inter or Roboto or system fonts
- A `/contribute` upload form (Harrison gathers media himself — no UI for guests)
- Any analytics, tracking pixels, social embeds

## 9. FINAL TONE GUIDANCE

Read this every iteration:

This site is a gift. Not a product, not a portfolio piece, not a template demonstration. Two people are getting married and the site is for them. Every choice should pass the test of: *"Would this still feel meaningful 30 years from now if they opened it?"*

Restraint over expression. Whitespace over content. One perfect line over five good ones. Their photos do the talking. The typography sets the tone. Everything else gets out of the way.

If anything feels clever — strip it.
If anything feels loud — quiet it.
If anything feels generic — replace it.
